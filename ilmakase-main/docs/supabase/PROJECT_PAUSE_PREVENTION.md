# Supabase 프로젝트 일시 정지 방지 가이드

## 🔍 문제 원인

**11MB는 용량 문제가 아닙니다!**

Supabase 무료 플랜에서는:
- **1주일 동안 비활성 상태**이면 자동으로 프로젝트가 일시 정지됩니다
- 데이터는 유지되지만 서비스가 중단됩니다
- 용량과 관계없이 활동 여부만 확인합니다

## ✅ 해결 방법

### 즉시 해결
1. Supabase 대시보드 접속
2. 일시 정지된 프로젝트 선택
3. **"Restore project"** 또는 **"Resume"** 버튼 클릭
4. 몇 분 후 프로젝트가 다시 활성화됩니다

## 🛡️ 방지 방법

### 방법 1: 정기적으로 접근하기
- **주 1회 이상** Supabase 대시보드에 접속
- 또는 애플리케이션을 사용하여 API 호출 발생

### 방법 2: 자동 핑 스크립트 (선택사항)
정기적으로 프로젝트에 요청을 보내는 스크립트를 설정할 수 있습니다.

### 방법 3: Pro 플랜으로 업그레이드
- Pro 플랜 ($25/월)은 자동 일시 정지가 없습니다
- 프로덕션 환경에서는 권장됩니다

## 📊 현재 상태

- **데이터베이스**: 11MB (매우 적음, 문제 없음 ✅)
- **제한**: 500MB까지 사용 가능
- **문제**: 비활성 상태로 인한 자동 일시 정지

## 💡 권장 사항

1. **개발 중**: 주 1회 이상 대시보드 접속
2. **프로덕션**: Pro 플랜 업그레이드 고려
3. **모니터링**: 정기적으로 프로젝트 상태 확인

## 🔔 알림 설정

Supabase 대시보드에서 프로젝트 일시 정지 알림을 설정할 수 있습니다:
- Settings → Notifications
- Project pause 알림 활성화
