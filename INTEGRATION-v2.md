# WorkLog + 일마카세 통합 프로젝트 v2.0

> 두 개의 독립적인 프로젝트(WorkLog, 일마카세)를 하나의 통합 서비스로 합치는 작업

---

## 🎯 프로젝트 목표

WorkLog의 **업무 기록 기능**과 일마카세의 **AI 분석 기능**을 결합하여,
**"메모장처럼 쓴 업무 기록이 이직용 경력기술서로 자동 변환되는"** 완전한 서비스를 만듭니다.

### 핵심 차별화 포인트
✅ **프로젝트별 자동 집계** - 업무를 프로젝트 단위로 자동 정리  
✅ **경력기술서 자동 생성** - 바로 복붙 가능한 이력서 문구 제공  
✅ **메모장 같은 UX** - 부담 없이 쓰고 AI가 알아서 정리  
✅ **월별 트렌드 분석** - 내 업무 패턴 시각화

---

## 📂 핵심 데이터 구조 (3단계)

```
프로젝트 (자동 매칭 or 수기 등록)
  └── 태스크 (수기 등록)
       └── 업무 (수기 등록, 메모장처럼)
```

### 실제 예시:

```
📦 프로젝트: 도매 플랫폼 런칭
   ├── 🎯 태스크: 3개 시스템 기획
   │    ├── ✅ 업무: 관리자 어드민 요구사항 정리 (완료)
   │    ├── ✅ 업무: 판매자 대시보드 플로우 작성 (완료)
   │    └── ⏳ 업무: 구매자 앱 화면 정의서 작성 (진행중)
   │
   ├── 🎯 태스크: API 명세서 검토
   │    ├── ✅ 업무: 결제 API 3건 리뷰 (완료)
   │    └── ✅ 업무: 주문 API 보안 이슈 발견 (완료)
   │
   └── 🎯 태스크: 개발사 커뮤니케이션
        ├── ✅ 업무: 주간 미팅 아젠다 작성 (완료)
        └── ⏳ 업무: 일정 지연 이슈 논의 (진행중)
```

## 📂 현재 상태

### WorkLog (업무 관리 앱)
```
WorkLog/
├── 데일리 업무 기록
├── 캘린더 뷰
├── 업무 목록 관리
├── 완료율 트래킹
└── 프로젝트 아카이브
```

**강점:**
- ✅ 깔끔한 UI/UX
- ✅ 업무 기록 인터페이스 완성
- ✅ 체크리스트 + 완료율 시스템

**부족한 점:**
- ❌ AI 분석 없음
- ❌ 프로젝트별 자동 집계 없음
- ❌ 경력기술서 생성 없음

---

### 일마카세 (AI 분석 서비스)
```
일마카세/
├── 오늘 업무 AI 한줄 평가
├── 5일치 패턴 분석
├── 프로젝트 카드 생성
├── 사고방식 분석
└── 인사이트 제공
```

**강점:**
- ✅ AI 기반 업무 평가
- ✅ 패턴 분석 로직
- ✅ 사고방식 분석
- ✅ 인사이트 제공

**부족한 점:**
- ❌ 업무 기록 UI 약함
- ❌ 캘린더 뷰 없음
- ❌ 완료율 트래킹 없음
- ❌ 프로젝트 구조 없음

## 🎨 통합 후 서비스 구조

```
일마카세 2.0 (통합 서비스)
│
├── 1. 업무 기록 (WorkLog 기반)
│   ├── 오전: 오늘 할 일 작성
│   │   - 프로젝트 자동 매칭 또는 수기 입력
│   │   - 태스크 입력
│   │   - 업무 메모장처럼 작성
│   │
│   └── 오후: 완료율 체크 + 상세 기록
│       - 체크박스 체크
│       - 진척도 슬라이더 (0~100%)
│       - 상세 내용 메모장처럼 작성
│       - AI 분석 받기
│
├── 2. AI 분석 (일마카세 AI)
│   ├── 일일 분석: 오늘 사고방식 평가
│   ├── 주간 분석: 5일치 패턴 (선택)
│   └── 월간 분석: 트렌드 + 회고 (유료)
│
├── 3. 프로젝트 카드 (이직/평가용)
│   ├── 프로젝트별 업무 자동 집계
│   ├── 태스크별 업무 정리
│   ├── 사고방식 분석
│   └── 경력기술서 문구 자동 생성
│       - 간결형 (200자)
│       - 상세형 (500자, 태스크별)
│       - STAR 형식
│
└── 4. 인사이트 (프리미엄)
    ├── 월별 트렌드 분석
    ├── 업무 효율 팁
    ├── 이직 준비 가이드
    └── 면접 시뮬레이션
```

## 🔄 통합 워크플로우

### Case 1: 일일 업무 기록 및 완료율 체크

```
[오전 출근]
    ↓
[일마카세 열기 - 오늘 할 일 작성]
    - 프로젝트: "도매 플랫폼 런칭" (자동 매칭)
    - 태스크: "3개 시스템 기획"
    - 업무: "관리자 어드민 요구사항 정리" (메모장처럼)
    ↓
[저장]
    ↓
[오후 퇴근 전]
    ↓
[일마카세 열기 - 오늘 한 일 체크]
    - [X] 관리자 어드민 요구사항 정리 (100%)
    - 상세: "요구사항 30개 정리, 보안 이슈 2건 발견"
    ↓
[AI 분석 받기]
    ↓
[AI 피드백 확인]
    - ✅ "보안 이슈 선제 발견 - 리스크 관리 사고"
    - ⚠️ "다음엔 우선순위도 함께 정리해보세요"
```

**AI 분석 횟수: 1회 사용 (무료 3회/월)**

---

### Case 2: 한달치 회고

```
[월말]
    ↓
[일마카세 > 회고]
    ↓
[한 달 요약 자동 생성]
    - 총 업무 일수: 20일
    - 완료율 평균: 78%
    - 주요 프로젝트: 도매플랫폼 (80%), 내부시스템 (20%)
    ↓
[AI 트렌드 분석]
    - 업무 유형: 기획 40%, 커뮤니케이션 35%, QA 25%
    - 월별 비교: 완료율 70% → 78% (↗ 8%p)
    - 인사이트: "기획 비중 증가 → 다음 달엔 실행 집중"
    ↓
[회고 가이드 (AI 질문)]
    Q: 이번 달 가장 아쉬웠던 점은?
    Q: 다음 달 가장 집중하고 싶은 것은?
    Q: 구체적 액션 3가지는?
```

**한달치 리포트: 유료 기능**

---

### Case 3: 이직/평가 준비 (프로젝트 카드)

```
[이직 준비 시작]
    ↓
[일마카세 > 프로젝트 카드]
    ↓
[프로젝트 선택: "도매 플랫폼 런칭"]
    ↓
[프로젝트 카드 자동 생성]
    - 기간: 2025.10~12 (3개월)
    - 역할: PM (총괄 기획)
    - 태스크별 업무 집계:
      • 3개 시스템 기획 (45개 업무)
      • API 명세서 검토 (28개 업무)
      • 개발사 커뮤니케이션 (32개 업무)
    ↓
[사고방식 분석]
    - 리스크 선제 관리
    - 협업 주도형
    - 데이터 기반 의사결정
    ↓
[경력기술서 문구 생성 (3가지 버전)]
    - 간결형 (200자): 바로 복붙 가능
    - 상세형 (500자): 태스크별 상세
    - STAR 형식: 면접 대비
    ↓
[복사하기] or [이력서에 추가]
```

**경력기술서 생성: 무료 1회/월, 유료 무제한**

```sql
-- 프로젝트 테이블
CREATE TABLE projects (
  id SERIAL PRIMARY KEY,
  user_id INT NOT NULL,
  name VARCHAR(200) NOT NULL,
  status VARCHAR(20) DEFAULT '진행중', -- '진행중', '완료', '보류'
  start_date DATE,
  end_date DATE,
  auto_matched BOOLEAN DEFAULT false,  -- 자동 매칭 여부
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

-- 태스크 테이블
CREATE TABLE tasks (
  id SERIAL PRIMARY KEY,
  project_id INT REFERENCES projects(id),
  name VARCHAR(200) NOT NULL,
  created_at TIMESTAMP DEFAULT NOW()
);

-- 업무 기록 테이블 (WorkLog + 일마카세 통합)
CREATE TABLE work_logs (
  id SERIAL PRIMARY KEY,
  user_id INT NOT NULL,
  task_id INT REFERENCES tasks(id),
  
  -- 기본 정보
  content TEXT NOT NULL,              -- "관리자 어드민 요구사항 정리"
  detail TEXT,                        -- 상세 내용
  work_date DATE NOT NULL,
  
  -- 완료율
  progress INT DEFAULT 0,             -- 0~100
  is_completed BOOLEAN DEFAULT false,
  
  -- AI 분석 결과 (JSON)
  ai_analysis JSONB,                  -- {"evaluation": "...", "suggestions": [...]}
  
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

-- 경력기술서 테이블
CREATE TABLE career_documents (
  id SERIAL PRIMARY KEY,
  user_id INT NOT NULL,
  project_id INT REFERENCES projects(id),
  
  -- 생성된 문구
  brief_version TEXT,                 -- 간결형 (200자)
  detailed_version TEXT,              -- 상세형 (500자)
  star_version TEXT,                  -- STAR 형식
  
  -- 메타 정보
  thinking_analysis JSONB,            -- 사고방식 분석
  generated_at TIMESTAMP DEFAULT NOW()
);

-- 월간 회고 테이블
CREATE TABLE monthly_reviews (
  id SERIAL PRIMARY KEY,
  user_id INT NOT NULL,
  year_month VARCHAR(7) NOT NULL,     -- "2025-12"
  
  -- 통계
  total_work_days INT,
  avg_completion_rate DECIMAL(5,2),
  work_type_distribution JSONB,      -- {"기획": 40, "커뮤니케이션": 35, ...}
  
  -- AI 인사이트
  ai_insights JSONB,
  
  -- 사용자 회고
  user_reflection TEXT,
  
  created_at TIMESTAMP DEFAULT NOW()
);

-- 인덱스
CREATE INDEX idx_work_logs_user_date ON work_logs(user_id, work_date);
CREATE INDEX idx_work_logs_task ON work_logs(task_id);
CREATE INDEX idx_projects_user ON projects(user_id);
CREATE INDEX idx_tasks_project ON tasks(project_id);
```

```javascript
// frontend/components/WorkLog/DailyLog.jsx

const handleAutoMatch = async (inputText) => {
  const response = await fetch('/api/worklog/auto-match', {
    method: 'POST',
    body: JSON.stringify({ content: inputText })
  });
  
  const { matched, project, alternatives } = await response.json();
  
  if (matched) {
    setSelectedProject(project);
    showToast(`📦 '${project.name}'으로 자동 매칭됐어요`);
  } else if (alternatives.length > 0) {
    showProjectSelector(alternatives);
  } else {
    suggestNewProject(inputText);
  }
};

const handleAIAnalysis = async () => {
  const response = await fetch('/api/ai-analysis/daily', {
    method: 'POST',
    body: JSON.stringify({
      workLogId: currentWorkLog.id,
      content: currentWorkLog.content,
      detail: currentWorkLog.detail
    })
  });
  
  const analysis = await response.json();
  setAIFeedback(analysis);
  
  // AI 분석 횟수 차감
  decrementAICredits();
};
```

### Q1. 기존 WorkLog/일마카세 데이터는?

**A.** 신뢰도 0.8 이상일 때만 자동 매칭, 그 외엔 선택지 제공.

**A.** 주 1~2회 분석하면 충분. 매일 할 필요는 없음.

**A.** 실제 업무 기록 기반이므로 구체적이고 신뢰도 높음.

### Q5. 통합 작업 시간은?
